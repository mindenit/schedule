## What UI Thing is

UI Thing is this project's component library. It is **not** an npm package — components are
scaffolded via CLI directly into `app/components/Ui/` as owned, editable source files. Built on
**Reka UI + Tailwind CSS v4**, Nuxt-native, inspired by shadcn/ui.

The entire component catalog lives at **https://uithing.com/components** — always check there
for up-to-date component names, props, and examples. Source for every component is on GitHub:
`https://github.com/BayBreezy/ui-thing/tree/v2/app/components/Ui/<ComponentName>`

## CLI — always use this, not copy-paste

All component work goes through `npx ui-thing@latest`. The CLI handles scaffolding, dep install,
and `nuxt.config` updates automatically.

```bash
# One-time project setup
npx ui-thing@latest init

# Add one or more components (interactive selector if no names given)
npx ui-thing@latest add button dialog select

# Add a shadcn theme (⚠ overwrites tailwind.css)
npx ui-thing@latest theme

# Add the defineShortcuts composable
npx ui-thing@latest shortcuts

# Add opinionated prettier config (uses tv in tailwindFunctions)
npx ui-thing@latest prettier
```

For full CLI flags and options → see `references/cli.md`

## Project conventions (critical)

**Component location:** `app/components/Ui/`
**Auto-import prefix:** `Ui` — all components are used as `<UiButton>`, `<UiDialog>`, `<UiSelect>`, etc.
**Styling:** uses `tailwind-variants` (`tv()`), NOT `clsx`/`cn()`/CVA
**Tailwind CSS file:** `app/assets/css/tailwind.css`
**Composables:** `app/composables/` | **Plugins:** `app/plugins/` | **Utils:** `app/utils/`

The `Ui` prefix is auto-generated by Nuxt's component discovery from the `app/components/Ui/`
directory — you never import components manually.

```vue
<!-- ✓ correct — Nuxt auto-imports with Ui prefix -->
<UiButton variant="default">Click me</UiButton>
<UiDialog v-model:open="isOpen">...</UiDialog>

<!-- ✗ wrong — no manual imports needed -->
<Button>Click me</Button>
import { Button } from "~/components/ui/button"
```

## Styling with tailwind-variants (`tv`)

UI Thing uses `tv()` from the `tailwind-variants` package, not CVA or `cn()`. When customizing
a scaffolded component or building a new one that follows the same pattern:

```ts
import { tv, type VariantProps } from "tailwind-variants"

const button = tv({
  base: "inline-flex items-center justify-center rounded-md text-sm font-medium",
  variants: {
    variant: {
      default: "bg-primary text-primary-foreground",
      outline: "border border-input bg-background",
      ghost: "hover:bg-accent hover:text-accent-foreground",
    },
    size: {
      default: "h-9 px-4",
      sm: "h-8 px-3",
      lg: "h-10 px-8",
      icon: "h-9 w-9",
    },
  },
  defaultVariants: { variant: "default", size: "default" },
})

type ButtonVariants = VariantProps<typeof button>
```

`cn()` (from `app/utils/cn.ts`) still exists in the project for one-off class merges in
non-UI-Thing components like `SidebarEvent.vue`, but it is **not** the styling mechanism for
Ui/ components.

## Composables shipped with UI Thing

After running `npx ui-thing@latest shortcuts`:

```ts
// Keyboard shortcuts
defineShortcuts({
  meta_k: { handler: () => openSearch() },       // ⌘K on mac, Ctrl+K on others
  ctrl_shift_f: { handler: () => toggleFilter() }, // combination
  "g-d": { handler: () => goToDashboard() },       // sequence (g then d)
  escape: () => closeDialog(),                      // simple form
})

// OS-specific key symbol
const { getKbdKey } = useKbd()
const meta = computed(() => getKbdKey("meta")) // → "⌘" or "Ctrl"

// Toast notifications (built-in with Sonner component)
useSonner("Success!", { description: "Schedule saved." })
useSonner.error("Failed", { description: err.message })
```

## Migrating from shadcn-nuxt (this project's current state)

This project is migrating from `shadcn-nuxt` to UI Thing. Key differences to keep in mind:

| Aspect | Old (shadcn-nuxt) | New (UI Thing) |
|---|---|---|
| Component location | `app/components/ui/<name>/` | `app/components/Ui/` |
| Usage in templates | `<Button>`, `<Dialog>` | `<UiButton>`, `<UiDialog>` |
| Styling | `cn()` + CVA (`buttonVariants`) | `tv()` from tailwind-variants |
| Distribution | `shadcn-nuxt` module + CLI | `ui-thing` CLI only |
| Config | `components.json` + `shadcn` in nuxt.config | `ui-thing.config.ts` |

**Known migration-specific issues in this codebase** (fix before/during migration):
- `app/components/AuthorsDialog.vue:43` — uses `buttonVariants({ variant: 'ghost', size: 'icon' })`
  from the old button. Replace with `<UiButton variant="ghost" size="icon" as-child>` wrapping
  the `<NuxtLink>`.
- `app/components/SidebarEvent.vue:24,41` — uses `cn()` for event-type color classes. Convert
  to a plain `:class` array binding (Vue handles this natively).
- `app/components/links/DataTable.vue` — manually imports Button, Checkbox, Input, DropdownMenu*,
  Table*, Badge from old `~/components/ui/*` paths. These imports break when the Ui/ directory
  is replaced. Remove explicit imports and rely on Nuxt auto-imports after scaffolding the
  equivalent Ui components.

For deeper migration guidance → see `references/migration.md`
